# Beaker Case Study: Joint Culture Plate Setup Gap

## Case Summary

During Epic Beaker microbiology workflow validation, the joint culture workflow was reviewed for default plate setup configuration.

A discrepancy was identified where MacConkey agar was not included in the default plate setup for joint aerobic cultures, despite being standard in comparable wound, tissue, and body fluid culture workflows.

## Workflow Area

- Epic Beaker Microbiology
- Joint culture workflow
- Plate setup configuration
- Aerobic culture media standardization

## Observation

MacConkey agar was available as an add-on but was not included in the default plate setup for joint cultures.

Comparable workflows where MacConkey is used as standard include:

- Wound cultures
- Tissue cultures
- Body fluid cultures

Joint cultures function as a subcategory of sterile site fluid/tissue culture workflows, so the absence of MacConkey created a workflow standardization concern.

## Potential Impact

- Inconsistent plate setup across similar aerobic culture workflows
- Increased reliance on manual add-ons
- Potential variability in Gram-negative organism recovery
- Reduced consistency across microbiology culture workflows
- Additional manual review burden for laboratory staff

## Analyst Assessment

This issue was not a system-breaking defect, but it represented a meaningful workflow standardization gap.

The concern was that similar specimen workflows should follow consistent baseline media setup logic unless there is a clear clinical or operational reason to differ.

## Recommendation

Recommend adding MacConkey agar to the default plate setup for joint aerobic culture workflows to align with established wound, tissue, and body fluid culture standards.

## Action Taken

A Beaker validation ticket was submitted requesting review of the joint culture default plate setup and consideration for adding MacConkey agar as a standard plate.

## Analyst Reflection

This case highlights the importance of validating not only whether an order can be placed, but whether the downstream workflow reflects established laboratory practice.

Small build differences, such as whether a plate is defaulted or only available as an add-on, can create variation in workflow execution and reduce standardization across benches.
